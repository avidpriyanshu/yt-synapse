## Context on YouTube Watch Page HTML

- Duration can be found via regex on watch page: `"lengthSeconds":"(\d+)"`
- View count can be extracted: `"viewCount":"(\d+)"` or via DOM structure

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (clipping file) | Untrusted: filename, URL, frontmatter content |
| YouTube watch page HTML | Untrusted: externally sourced, may change structure |
| Duration/viewCount extraction | Untrusted: regex-extracted values from HTML, may be malformed |
| Generated markdown | Trusted: written to vault, but should be well-formed YAML |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | **I** (Information Disclosure) | View count display | **Accept** | View counts are public on YouTube; no PII |
| T-01-02 | **I** (Information Disclosure) | Thumbnail URL formula | **Accept** | Thumbnail URLs are public; hqdefault is standard YouTube endpoint |
| T-01-03 | **T** (Tampering) | YouTube HTML scraping | **Mitigate** | Validate regex matches before using (non-empty, numeric for counts); use try-catch on HTML parsing |
| T-01-04 | **D** (Denial of Service) | fetchVideoMetadata() network calls | **Mitigate** | Set fetch timeout (not implemented in current engine.js, but log failures gracefully); rate-limit by batch size (VIDEO_BATCH = 15) |
| T-01-05 | **T** (Tampering) | Markdown generation (YAML injection) | **Mitigate** | Use existing escapeYamlString() for all user-controlled fields; validate view count as integer before embedding |
| T-01-06 | **R** (Repudiation) | No logging of metadata extraction success/failure | **Mitigate** | Log duration/viewCount enrichment attempts via logPhase pattern (already established) |

**Acceptance Rationale:**
- View counts and thumbnails are public data; no confidentiality risk
- Generation targets local vault; no network exposure of generated content
- HTML scraping already used for channelId; metadata extraction uses same pattern

**Mitigation Details:**
- All regex matches validated as non-empty before use (extract, then validate)
- Number fields (duration, viewCount) cast to integers and range-checked (>= 0)
- Graceful degradation: extraction failures logged but don't block note generation
- YAML injection prevented by existing escapeYamlString() utility
</threat_model>
