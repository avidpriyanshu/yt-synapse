## Deviations from Plan

### Auto-adapted: No watcher/extractor/resolver/harvester/generator files existed

The plan was written against an idealized architecture with 7 distinct files. The actual codebase had those functions consolidated into fewer files (main.js, engine.js, processor.js). The reorganization was adapted to the actual files present:
- main.js handles watcher + extractor + generator logic (stays at root)
- engine.js handles resolver logic (stays at root)
- processor.js maps to processing/ folder

### Auto-fixed: ecosystem.config.js path updates

The plan did not mention that ecosystem.config.js references to server.js and log paths needed updating when the file moved from logic-engine/ to logic-engine/config/. Fixed automatically (Rule 1).

### Auto-fixed: logger.js __dirname depth

logger.js used `path.dirname(__dirname)` to find the project root. After moving from logic-engine/ to logic-engine/utils/, this needed to become `path.resolve(__dirname, '..', '..')`. Fixed automatically (Rule 1).

### Out of scope: topic-blacklist.json and topic-remap.json

These data files at `.planning/topic-blacklist.json` and `.planning/topic-remap.json` are referenced by processor.js and are not planning docs — left at `.planning/` root as they were. No change needed.
