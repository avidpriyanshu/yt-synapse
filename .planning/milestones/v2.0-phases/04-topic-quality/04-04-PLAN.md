# Phase 4 Plan 4: Topic Quality - Implementation

**Objective:** Replace compromise NLP library with regex-based extraction, improve topic quality, add deduplication utility, and externalize topic blacklist configuration.

**Requirements Addressed:**
- TOPIC-01: System replaces compromise NLP library with higher-quality alternative

### Task 3: Improve topic extraction (replace compromise dependency)
**Type:** auto
**Effort:** Medium, core logic refactor

Replace compromise NLP library with regex-based noun phrase extraction. No external NLP dependency needed.

**Behavior:**
- Remove `const nlp = require('compromise')` from processor.js
- Implement regex-based extraction for:
  1. Capitalized noun phrases (2-3 words): "Machine Learning", "Web Development"
  2. ALL CAPS acronyms (2-10 chars): "AI", "HTML", "NASA"
  3. Named entities from title patterns (places, people mentioned)
- Keep existing heuristics:
  - Reject single-word topics unless they're acronyms or 5+ chars
  - Reject topics ending with prepositions
  - Apply length filters (3-50 chars)
- Improve by reducing false positives from weak compromise detection

**Success Criteria:**
1. processor.js has no `require('compromise')` statement
2. `extractCandidateTopics()` uses regex-based extraction
3. Test cases pass: "Machine Learning" extracted, "You Can" filtered
4. Output quality verified on sample titles (fewer noise words)

**Files to create/modify:**
- `logic-engine/processor.js` (rewrite extraction logic, remove compromise dependency)
- `package.json` (remove compromise dependency if possible)
