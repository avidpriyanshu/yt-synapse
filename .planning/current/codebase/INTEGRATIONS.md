## APIs & External Services

  - Location: `/Users/avid/Desktop/yt-vault/logic-engine/engine.js` (lines 46-59)

  - URL: `https://www.youtube.com/feeds/videos.xml?channel_id={channelId}`
  - Location: `/Users/avid/Desktop/yt-vault/logic-engine/engine.js` (line 73-115)

  - Client: `rss-parser` package (v3.13.0)
  - Auth: None (public RSS feed)
  - Returns: Feed title, items with `yt:videoId`, `yt:channelId`, title, link, pubDate

## Data Storage

**Local Persistent Storage:**
- **`channels.json`** - Channel metadata cache
  - Location: `/Users/avid/Desktop/yt-vault/logic-engine/channels.json`
  - Structure: `{ channels: Record<channelId, { resolvedAt, sourceVideoId, feedTitle, label? }> }`
  - Purpose: Cache resolved channel info to avoid repeated lookups
  - Updated by: `upsertChannelRecord()` in `main.js` (line 75-83)

- **`history.json`** - Processed video IDs
  - Location: `/Users/avid/Desktop/yt-vault/logic-engine/history.json`
  - Structure: `{ videoIds: string[] }`
  - Purpose: Prevent duplicate video note generation
  - Updated by: `appendHistoryAtomic()` in `main.js` (line 66-73)

**Obsidian Vault (Local Filesystem):**
- **Generated Video Notes** - Markdown files
  - Location: `{VAULT_ROOT}/videos/{title}-{videoId}.md`

- **Generated Topic Index Pages** - Markdown files
  - Location: `{VAULT_ROOT}/topics/{topicLabel}.md`

- **Clipping Input Directory** - Markdown files (user-created)
  - Location: `{VAULT_ROOT}/clippings/`
  - Watched by: `chokidar` file system watcher in `main.js`
  - Format: YAML frontmatter with YouTube URL in `source` field

## Obsidian Integration

**Obsidian Vault Coupling:**
- **Dataview Plugin** - Required for dynamic queries
  - Usage: Topic index pages use `TABLE` queries to list videos
  - Example in `buildTopicMarkdown()` in `main.js` (line 220-225)
  - Syntax: `FROM "videos" WHERE contains(file.outlinks, this.file.link)`

- **Wikilink System** - Knowledge graph connections
  - Channel wikilinks: `[[Channel Name]]` in video note body
  - Topic wikilinks: `[[Topic]]` in both frontmatter and body
  - Generated by: `channelWikilink` and `topicLinksLine` in `buildVideoMarkdown()` (main.js, line 184, 175)

- **File Structure Expectations:**
  - Root vault directory structure:
    - `videos/` - Generated video notes
    - `topics/` - Generated topic indices
    - `clippings/` - User input files
    - `channels/` - Optional channel index pages (referenced but not auto-created in current code)
    - `.obsidian/` - Obsidian config (includes plugin settings)

## Environment Configuration

**Required Environment Variables:**
- `VAULT_ROOT` - Path to Obsidian vault directory
  - Default: `../obsidian-vault` (relative to logic-engine)
  - Example: `export VAULT_ROOT=/Users/username/Documents/MyVault`

**Optional Environment Variables:**
- `CHOKIDAR_POLLING` - Force polling mode
  - Values: '1' or 'true'
  - Use case: When native file system events fail (network drives, Docker)

**Secrets Management:**
- No API keys, tokens, or credentials required
- All external access is to public YouTube endpoints

## Batch Processing Configuration

**Video Batch Limit:**
- `VIDEO_BATCH = 15` constant in `main.js` (line 11)
- Purpose: Limit new videos processed per clipping trigger
- Prevents: Overwhelming the vault with too many notes on first sync

**File Watcher Debounce:**
- Debounce delay: 800ms in `main.js` (line 368)
- Purpose: Prevent multiple triggers for the same file write
- Write finish stabilization: 400ms threshold + 100ms polling interval

## Webhooks & Callbacks

**Incoming:**
- None (file system watcher-triggered, not webhook-triggered)

**Outgoing:**
- None (read-only access to YouTube, write-only to local vault)

## Data Flow Summary

1. **User creates** clipping file in `{VAULT_ROOT}/clippings/`
2. **Watcher detects** file change via `chokidar`
3. **Parser extracts** YouTube video/channel ID from frontmatter or body
4. **Engine resolves** channel ID by fetching video page HTML and regex-matching
5. **Engine fetches** YouTube RSS feed for channel
6. **Processor extracts** topics from video titles using NLP (compromise)
7. **Generator creates** video markdown notes with wikilinks
8. **Generator creates** topic stub pages if new topics found
9. **History tracker** records video IDs to prevent duplicates
10. **Channel tracker** caches channel metadata to avoid repeated lookups
11. **Health check** can verify all wikilinks have corresponding .md files
